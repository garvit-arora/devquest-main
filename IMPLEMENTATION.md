# DevQuest AI Implementation Notes

## Product Flow

DevQuest AI now centers on one flow:

1. Sign in with GitHub.
2. View owner-approved repositories in Projects.
3. Star a repository on GitHub.
4. Click Verify star.
5. Receive 500 prompt credits once per repository.
6. Create a DevQuest API key.
7. Call `https://api.devquest.ai/v1`.
8. The backend blocks access when no approved repository entitlement remains verified.

## Backend

- `app.py` creates the FastAPI app and registers routers.
- `config.py` owns environment-backed settings.
- `state.py` contains the current in-memory stores and is the seam to replace with PostgreSQL repositories.
- `deps.py` contains shared FastAPI dependencies such as session auth.
- `routers/` owns HTTP endpoints by product area.
- `services/` owns GitHub entitlement and gateway validation logic.
- GitHub OAuth uses `/api/auth/github/login` and `/api/auth/github/callback`.
- Approved repositories are read from `DEVQUEST_APPROVED_REPOS`.
- Star verification uses `GET /user/starred/{owner}/{repo}` with the authenticated GitHub token.
- Repository rewards are written to the immutable ledger with idempotency keys.
- API keys are generated by the backend, hashed with `DEVQUEST_API_KEY_PEPPER`, and never stored raw.
- Gateway requests validate repository entitlement, credits, model access, rate limits, input size, and output size before calling Azure.
- Sponsor submissions save server-side and send Azure Communication Services Email when configured.
- Worker loop rechecks entitlements every 10 minutes.

## Azure Services

- Azure Container Apps: target for web, API, and worker.
- Azure OpenAI / Azure AI Foundry: upstream model provider through DevQuest aliases.
- Azure Database for PostgreSQL: required production persistence target.
- Azure Service Bus: verification job queue hook.
- Azure Key Vault: secret retrieval helper.
- Azure Application Insights: telemetry infrastructure.
- Azure Communication Services Email: sponsor notifications.
- Azure Container Registry: deployment image target.

## Required Owner Data

- Real GitHub OAuth credentials.
- Real approved repository list in `DEVQUEST_APPROVED_REPOS`.
- Azure model endpoint, API key, and deployment names.
- Azure Communication Services Email sender and owner destination email.
- Azure Service Bus connection string and queue name.
