## 1. Open config.toml

Open PowerShell and run:

```powershell
notepad "$env:USERPROFILE\.codex\config.toml"
```

If Notepad asks to create the file, choose **Yes**.

The file location is:

```text
C:\Users\<USERNAME>\.codex\config.toml
```

## 2. Paste the DevQuest provider

Paste this into `config.toml`, then save the file:

```toml
model = "gpt-5.6-sol"
model_provider = "devquest"
model_reasoning_effort = "medium"

[model_providers.devquest]
name = "DevQuest"
base_url = "https://devquest.garvitarora.xyz/v1"
env_key = "DEVQUEST_API_KEY"
wire_api = "responses"
```

In the IDE extension, you can also open the same file from Gear icon, Codex Settings, Open config.toml.

## 3. Set your API key

In PowerShell, set your DevQuest API key for the current terminal:

```powershell
$env:DEVQUEST_API_KEY = "dq_live_your_key"
```

Then save it for future terminals:

```powershell
setx DEVQUEST_API_KEY "dq_live_your_key"
```

Use the API key shown once in the DevQuest dashboard when you create a key.

## 4. Restart your editor

Close and reopen VS Code, Cursor, Windsurf, or your VS Code fork. This reloads the `DEVQUEST_API_KEY` environment variable and the updated `config.toml`.

## 5. Launch Codex

After restarting, open the Codex extension or run:

```powershell
codex
```

## What DevQuest supports

DevQuest routes model inference through:

```text
Codex client -> DevQuest API -> model provider
```

Local code edits, terminal commands, file inspection, diffs, and approval prompts still happen on the user's computer.

The gateway implements `POST /v1/responses` for Codex-compatible clients, including text responses, SSE text streaming, non-streaming function-call output items, function-call output ingestion, usage reporting, credit checks, and repository-star enforcement. Streaming tool-call deltas should be treated as experimental until validated with your exact Codex client version. Use medium reasoning for the DevQuest Codex provider.

Use the model enabled on your API key. The dashboard currently offers `DeepSeek-V4-Pro`, `gpt-5.5`, `gpt-5.6-luna`, and `gpt-5.6-sol`, so this guide uses `gpt-5.6-sol`.

## Not included

DevQuest API keys do not include ChatGPT-plan usage, Codex Cloud tasks, OpenAI-hosted remote agents, cloud delegation, or proprietary features outside DevQuest-supported models.
