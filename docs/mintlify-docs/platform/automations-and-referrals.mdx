## Automations

DevQuest Automations let users build n8n-style flows with triggers, AI actions, app calls, and storage nodes.

Supported trigger starters include:

- Sheet row added
- Form submitted
- Waitlist email received
- Webhook received
- Scheduled run
- Manual run

The visible workflow canvas focuses on app triggers, AI steps, and actions. Repository access is enforced before sensitive actions run.

## AI tasks

Use a DevQuest AI node when a workflow should classify, summarize, enrich, score, or route incoming data.

Each run uses DevQuest credits and is limited by the user's active entitlement. The default thinking option exposed in the dashboard is `medium`.

## Database save nodes

Database save nodes can store workflow output after an AI step or app action.

When a database credential is added, DevQuest stores only a protected credential record. Raw secrets are not returned to the dashboard.

<Warning>
  Only add credentials that your workflow needs. Use encrypted secret storage for any reusable external password.
</Warning>

## Referrals

Use referral links in this format:

```text
https://devquest.space/r/USER_REFERRAL_CODE
```

When the link opens, DevQuest records the referral click, stores a secure HTTP-only cookie, and redirects the user to normal sign in without a visible referral query parameter.

Referral credits are awarded only after the referred user completes account connection successfully.

## Notifications

Dashboard notifications are stored for the user and can also be delivered by email when notification delivery is configured.
